# Функция заявки на блочно-модульные изделия

## Что было реализовано

### 1. Модальное окно с формой заявки

При клике на карточку "Блочно-модульные изделия" на главной странице открывается всплывающее окно с формой, содержащей:

- **ФИО** (обязательное поле)
- **Организация** (обязательное поле)
- **Должность** (обязательное поле)
- **Телефон** (обязательное поле)
- **Email** (обязательное поле)
- **Загрузка файла ТЗ** (обязательное поле, макс. 10 МБ)
  - Поддерживаемые форматы: PDF, DOC, DOCX, TXT, ZIP, RAR
- **Комментарий** (необязательное поле)

### 2. Обработка и сохранение данных

После отправки формы:
1. Проверяется размер файла (не более 10 МБ)
2. Файл сохраняется в папку `received_data/modular_requests/`
3. Данные заявки сохраняются в JSON файл
4. Отправляется email с заявкой (если настроен)

### 3. Отправка email

Автоматическая отправка email-уведомлений на указанный адрес с:
- Всеми данными заявителя
- Прикрепленным файлом ТЗ
- Временной меткой

## Использование

### Для пользователей

1. Откройте главную страницу приложения
2. Нажмите на карточку **"Блочно-модульные изделия"**
3. Заполните все обязательные поля формы:
   - Введите ФИО, организацию, должность
   - Укажите контактный телефон и email
   - Прикрепите файл с техническим заданием
   - Опционально добавьте комментарий
4. Нажмите **"Отправить заявку"**
5. Дождитесь подтверждения об успешной отправке

**Примечания:**
- Все поля, отмеченные красной звездочкой (*), обязательны для заполнения
- Размер файла не должен превышать 10 МБ
- Поддерживаются форматы: PDF, DOC, DOCX, TXT, ZIP, RAR

### Для администраторов

#### Настройка email-уведомлений

1. Откройте файл `email_config.py`
2. Настройте параметры SMTP сервера:
   ```python
   SMTP_SERVER = "smtp.gmail.com"
   SMTP_PORT = 587
   SENDER_EMAIL = "your-email@gmail.com"
   SENDER_PASSWORD = "your-app-password"
   RECEIVER_EMAIL = "recipient@example.com"
   ```
3. Установите `SEND_EMAIL = True`
4. Перезапустите сервер

Подробная инструкция: см. `EMAIL_SETUP_INSTRUCTIONS.md`

#### Просмотр полученных заявок

Все заявки сохраняются локально в папке:
```
app/received_data/modular_requests/
```

Файлы:
- `YYYYMMDD_HHMMSS_filename.ext` - загруженный файл ТЗ
- `YYYYMMDD_HHMMSS_data.json` - данные заявки

Пример содержимого JSON файла:
```json
{
  "timestamp": "2025-11-07T14:30:00",
  "fullname": "Иванов Иван Иванович",
  "organization": "ООО Компания",
  "position": "Инженер",
  "phone": "+7 (999) 123-45-67",
  "email": "ivanov@example.com",
  "comment": "Требуется консультация",
  "file_name": "technical_specification.pdf",
  "file_path": "received_data/modular_requests/20251107_143000_technical_specification.pdf",
  "file_size_mb": 2.45
}
```

## Технические детали

### API Endpoint

**POST** `/api/submit-modular-request`

**Параметры (multipart/form-data):**
- `fullname` (string, required)
- `organization` (string, required)
- `position` (string, required)
- `phone` (string, required)
- `email` (string, required)
- `comment` (string, optional)
- `tz_file` (file, required, max 10MB)

**Ответ (успех):**
```json
{
  "status": "success",
  "message": "Заявка успешно принята и сохранена. Email отправлен.",
  "request_id": "20251107_143000",
  "email_sent": true
}
```

**Ответ (ошибка):**
```json
{
  "status": "error",
  "message": "Размер файла превышает 10 МБ"
}
```

### Безопасность

- Проверка размера файла (макс 10 МБ)
- Фильтрация допустимых расширений файлов
- Безопасное формирование имен файлов с временной меткой
- Валидация всех обязательных полей
- UTF-8 кодирование для корректной обработки кириллицы

### Логирование

Все действия логируются:
- Получение заявки
- Сохранение файла и данных
- Отправка email
- Ошибки и предупреждения

Просмотр логов в консоли сервера.

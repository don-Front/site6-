# АЛГОРИТМ ПОДБОРА РАСХОДОМЕРОВ

## 1. ОБЩАЯ СТРУКТУРА СИСТЕМЫ

### 1.1. Типы оборудования
Система поддерживает три категории оборудования:
- **Расходомеры для газов** (`gas`)
- **Расходомеры для жидкостей** (`liquid`)
- **Денситометры** (`densitometer`)

### 1.2. База данных оборудования
Данные хранятся в файле `data.json` в следующей структуре:
```json
{
  "gas": [...],      // Массив расходомеров для газов
  "liquid": [...],   // Массив расходомеров для жидкостей
  "densitometer": [...]  // Массив денситометров
}
```

### 1.3. Карточки оборудования в системе

#### Для газов:
1. **Turbo Flow UFG-F-V** (id: `ufg2`)
2. **Turbo Flow TFG-S** (id: `tfgs1`)
3. **Turbo Flow TFG-H** (id: `tfgh1`)
4. **Turbo Flow UFG-F-C** (id: `ufgfc1`)
5. **Turbo Flow GFG** (id: `gfg2`)
6. **Turbo Flow UFG-H** (id: `ufgh1`)

#### Для жидкостей:
1. **Расходомер Turbo Flow CFM** (id: `cfm1`)
2. **Расходомер Turbo Flow UFL** (id: `ufl1`)

#### Денситометры:
1. **Turbo Flow UDM** (id: `udm1`)

## 2. ПАРАМЕТРЫ ПОИСКА

### 2.1. Обязательные параметры
- **Среда** (`medium`): `"gas"` или `"liquid"` — определяет, из какой категории выбирать оборудование

### 2.2. Опциональные параметры (используются для фильтрации)

#### Технические параметры:
1. **Расход** (`flow_min`, `flow_max`, `flow_unit`)
   - Единицы измерения:
     - Для газов: `"м³/ч"`, `"ст.м³/ч"` (стандартные м³/ч)
     - Для жидкостей: `"м³/ч"`, `"кг/ч"` (массовый расход)
   
2. **Давление** (`pressure_min`, `pressure_max`, `pressure_unit`)
   - Единицы: `"МПа"`, `"бар"` (автоматическая конвертация: 1 бар = 0.1 МПа)

3. **Температура измеряемой среды** (`temperature_min`, `temperature_max`, `temperature_unit`)
   - Единицы: `"°C"`

4. **Температура окружающей среды** (`ambient_temp_min`, `ambient_temp_max`)
   - Единицы: `"°C"`

5. **Точность** (`accuracy`)
   - В процентах (%)

6. **Диаметр** (`diameter`)
   - В миллиметрах (мм)

7. **Плотность** (`density_min`, `density_max`) — только для жидкостей
   - В кг/м³

### 2.3. Параметры для отображения (не влияют на фильтрацию)
- **Сфера применения** (`application`): `"industrial"`, `"utility"` и др.
- **Тип газа** (`gas_type`): `"natural"`, `"propane"`, `"air"` и др.

## 3. АЛГОРИТМ ФИЛЬТРАЦИИ

### 3.1. Шаг 1: Выбор категории оборудования
```
ЕСЛИ medium == "gas":
    equipment_list = data["gas"]
ИНАЧЕ ЕСЛИ medium == "liquid":
    equipment_list = data["liquid"]
ИНАЧЕ ЕСЛИ medium == "densitometer":
    equipment_list = data["densitometer"]
```

### 3.2. Шаг 2: Конвертация единиц измерения

#### 3.2.1. Конвертация стандартных м³/ч в рабочие м³/ч (для газов)
Если `flow_unit == "ст.м³/ч"`:
```
P_st = 101.325 кПа (стандартное давление)
T_st = 293.15 K (+20°C)
P_work = среднее(давление_мин, давление_макс) × 1000  // МПа → кПа
T_work = среднее(температура_мин, температура_макс) + 273.15  // °C → K

коэффициент_конвертации = (P_st / P_work) × (T_work / T_st)
расход_рабочий = расход_стандартный × коэффициент_конвертации
```

#### 3.2.2. Конвертация массового расхода в объемный (для жидкостей)
Если `medium == "liquid"` И `flow_unit == "кг/ч"`:
```
средняя_плотность = среднее(плотность_мин, плотность_макс)
расход_м3ч = расход_кгч / средняя_плотность
```

#### 3.2.3. Конвертация давления из бар в МПа
Если `pressure_unit == "бар"`:
```
давление_МПа = давление_бар × 0.1
```

### 3.3. Шаг 3: Проверка каждого оборудования

Для каждого оборудования из выбранной категории выполняется последовательная проверка по следующим критериям:

#### 3.3.1. Проверка давления
```
ЕСЛИ указано pressure_min ИЛИ pressure_max:
    spec_pressure_min = извлечь_мин_значение(equipment.specs.pressureRange)
    spec_pressure_max = извлечь_макс_значение(equipment.specs.pressureRange)
    
    ЕСЛИ pressure_min < spec_pressure_min:
        ОТКЛОНИТЬ (не подходит)
    ЕСЛИ pressure_max > spec_pressure_max:
        ОТКЛОНИТЬ (не подходит)
```

#### 3.3.2. Проверка температуры измеряемой среды
```
ЕСЛИ указано temperature_min ИЛИ temperature_max:
    spec_temp_min = извлечь_мин_значение(equipment.specs.tempRange)
    spec_temp_max = извлечь_макс_значение(equipment.specs.tempRange)
    
    ЕСЛИ temperature_min < spec_temp_min:
        ОТКЛОНИТЬ
    ЕСЛИ temperature_max > spec_temp_max:
        ОТКЛОНИТЬ
```

#### 3.3.3. Проверка температуры окружающей среды
```
ЕСЛИ указано ambient_temp_min ИЛИ ambient_temp_max:
    spec_ambient_min = извлечь_мин_значение(equipment.specs.ambientTempRange)
    spec_ambient_max = извлечь_макс_значение(equipment.specs.ambientTempRange)
    
    ЕСЛИ ambient_temp_min < spec_ambient_min:
        ОТКЛОНИТЬ
    ЕСЛИ ambient_temp_max > spec_ambient_max:
        ОТКЛОНИТЬ
```

#### 3.3.4. Проверка расхода
```
ЕСЛИ указано flow_min ИЛИ flow_max:
    spec_flow_min = извлечь_мин_значение(equipment.specs.flowRange)
    spec_flow_max = извлечь_макс_значение(equipment.specs.flowRange)
    
    ЕСЛИ flow_min < spec_flow_min:
        ОТКЛОНИТЬ
    ЕСЛИ flow_max > spec_flow_max:
        ОТКЛОНИТЬ
```

#### 3.3.5. Проверка точности
```
ЕСЛИ указано accuracy:
    spec_accuracy_min = извлечь_мин_значение(equipment.specs.accuracy)
    
    ЕСЛИ accuracy < spec_accuracy_min:
        ОТКЛОНИТЬ
    // Примечание: требуемая точность должна быть не хуже (не меньше) минимальной спецификации
```

#### 3.3.6. Проверка диаметра
```
ЕСЛИ указано diameter:
    spec_diameter_min = извлечь_мин_значение(equipment.specs.diameter)
    spec_diameter_max = извлечь_макс_значение(equipment.specs.diameter)
    
    ЕСЛИ diameter < spec_diameter_min ИЛИ diameter > spec_diameter_max:
        ОТКЛОНИТЬ
```

#### 3.3.7. Проверка плотности (только для жидкостей)
```
ЕСЛИ medium == "liquid" И (указано density_min ИЛИ density_max):
    spec_density_min = извлечь_мин_значение(equipment.specs.densityRange)
    spec_density_max = извлечь_макс_значение(equipment.specs.densityRange)
    
    ЕСЛИ density_min < spec_density_min:
        ОТКЛОНИТЬ
    ЕСЛИ density_max > spec_density_max:
        ОТКЛОНИТЬ
```

### 3.4. Шаг 4: Извлечение значений из диапазонов

Функции `_extract_min_value()` и `_extract_max_value()` обрабатывают строки диапазонов в различных форматах:

#### Поддерживаемые форматы:
1. **"X-Y"** или **"X–Y"** (с длинным тире): `"0,1-32 МПа"`
2. **"от X до Y"**: `"от 0,1 до 32 МПа"`
3. **"X...Y"**: `"-50…+70°C"`
4. **С запятыми вместо точек**: `"0,1-32"` → `0.1-32`

#### Алгоритм извлечения:
```
1. Заменить запятые на точки
2. Заменить длинные тире (–, —, …) на обычный дефис (-)
3. Удалить символ ±
4. Найти паттерн "от X до Y" → вернуть X (мин) или Y (макс)
5. Иначе найти паттерн "X-Y" → вернуть min(X,Y) (мин) или max(X,Y) (макс)
```

### 3.5. Шаг 5: Дополнительная фильтрация по ID

После основной фильтрации может применяться дополнительная фильтрация по разрешенным ID оборудования:
```
ЕСЛИ указан параметр allowedEquipmentIds:
    equipment_results = отфильтровать по ID из списка allowedEquipmentIds
```

## 4. ПРИМЕРЫ РАБОТЫ АЛГОРИТМА

### Пример 1: Подбор расходомера для газа
**Входные параметры:**
- Среда: `gas`
- Расход: 100-200 м³/ч
- Давление: 16-25 МПа
- Температура: -30 до +70°C
- Диаметр: 50 мм
- Точность: 1%

**Процесс:**
1. Выбирается категория `gas` → 6 расходомеров
2. Для каждого проверяется:
   - Давление: 16-25 МПа входит в диапазон спецификации?
   - Температура: -30 до +70°C входит в диапазон?
   - Расход: 100-200 м³/ч входит в диапазон?
   - Диаметр: 50 мм входит в диапазон?
   - Точность: 1% не хуже минимальной спецификации?

**Результат:** Только те расходомеры, которые прошли все проверки

### Пример 2: Подбор расходомера для жидкости с массовым расходом
**Входные параметры:**
- Среда: `liquid`
- Расход: 1000-2000 кг/ч
- Плотность: 800-900 кг/м³
- Давление: 1.6 МПа

**Процесс:**
1. Конвертация расхода: средняя плотность = 850 кг/м³
   - 1000 кг/ч → 1000/850 = 1.176 м³/ч
   - 2000 кг/ч → 2000/850 = 2.353 м³/ч
2. Выбирается категория `liquid` → 2 расходомера
3. Проверка по конвертированным значениям расхода и плотности

## 5. ОСОБЕННОСТИ РЕАЛИЗАЦИИ

### 5.1. Логика проверки диапазонов
- Если пользователь указал только минимальное значение параметра, проверяется только нижняя граница
- Если пользователь указал только максимальное значение, проверяется только верхняя граница
- Если указаны оба значения, проверяются обе границы

### 5.2. Обработка отсутствующих параметров
- Если параметр не указан, проверка по этому параметру пропускается
- Оборудование считается подходящим, если прошло все указанные проверки

### 5.3. Логирование
Система ведет подробное логирование процесса фильтрации:
- Параметры поиска
- Результаты проверки каждого оборудования
- Причины отсеивания неподходящего оборудования

### 5.4. Возврат результатов
Система возвращает:
- **Подходящее оборудование** — массив расходомеров, прошедших все проверки
- **Неподходящее оборудование** (опционально) — массив с указанием причин отсеивания

## 8. ЗАКЛЮЧЕНИЕ

Алгоритм подбора расходомеров основан на **строгой фильтрации по техническим параметрам**. Система проверяет соответствие каждого оборудования заданным пользователем условиям эксплуатации, автоматически конвертируя единицы измерения и извлекая значения из диапазонов спецификаций.

**Ключевые принципы:**
- Все проверки выполняются последовательно
- Оборудование отсеивается при несоответствии хотя бы одному параметру
- Конвертация единиц измерения выполняется автоматически
- Система гибко обрабатывает различные форматы записи диапазонов

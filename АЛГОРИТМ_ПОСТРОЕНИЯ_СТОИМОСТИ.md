# АЛГОРИТМ ПОСТРОЕНИЯ СТОИМОСТИ

Расчёт стоимости выполняется **на стороне клиента (в JavaScript)** на основе таблиц цен, зашитых в код. Для каждого типа расходомера — своя таблица и своя логика.

---

## 1. TURBO FLOW UFG-F-V

**Входные параметры:** DN, давление (МПа), точность (%)  
**Таблица цен:** `ufgfvPriceTable[DN][лучи][давление]`  
**Доступные DN:** 50, 65, 80, 100, 125, 150, 200, 250, 300, 350, 400, 450, 500, 600, 700, 800  
**Уровни давления:** 1.6, 6.3, 10, 16, 25 МПа

**Алгоритм:**
1. Если точного DN нет в таблице — берётся ближайший больший DN
2. По точности определяется количество лучей из ключей прайса для данного DN:
   - Точность >= 1% → **минимальный** ключ (наименьшее число лучей)
   - Точность < 1% → **максимальный** ключ (наибольшее число лучей)
3. По давлению выбирается столбец — берётся **следующий уровень выше** введённого давления (если давление = 6.3, берётся столбец 10). Если давление выше максимального (25) — берётся столбец 25.
4. Цена = `таблица[DN][лучи][давление]`
5. **Если точность < 1%** — итоговая цена умножается на **1,2** и округляется до целого рубля

**Отображение:** `UFG-F-V (DN100, 4 луча, 10 МПа, 1-2%)`

## 5. TURBO FLOW CFM

**Входные параметры:** DN, давление (МПа)  
**Таблица цен:** `cfmPriceTable[DN][давление]`  
**Доступные DN:** 15, 20, 25, 40, 50, 80, 100, 150, 200, 250  
**Уровни давления:** 1.6, 2.5, 4.0, 6.3, 10, 25 МПа

**Алгоритм:**
1. По давлению — берётся следующий уровень выше введённого. Если выше 25 — берётся 25.
2. Если точного DN нет — берётся ближайший больший DN
3. Цена = `таблица[DN][давление]`
4. **Особенность:** если цена = null (по запросу), система ищет цену:
   - Сначала в предыдущем (меньшем) DN при том же давлении
   - Если не нашлось — в соседнем (меньшем) столбце давления для того же DN

**Отображение:** `CFM (DN100, 10 МПа)`

## 8. ОБЩАЯ ЛОГИКА ВЫБОРА ДАВЛЕНИЯ

Для всех приборов (кроме UFG-F-C) действует одинаковая логика выбора столбца давления:

- Если введённое давление **строго меньше** уровня → берётся этот уровень
  - Пример: введено 5 МПа, уровни [1.6, 6.3, 10, 16] → берётся 6.3
- Если введённое давление **равно** уровню → берётся **следующий** уровень
  - Пример: введено 6.3 МПа → берётся 10
- Если введённое давление **выше максимального** → берётся максимальный уровень
  - Пример: введено 30 МПа, макс. 25 → берётся 25

**Логика:** нельзя продавать расходомер с давлением, равным рабочему — нужен запас.

---

## 9. ОБЩАЯ ЛОГИКА ПОИСКА DN

Если введённый DN отсутствует в таблице — берётся **ближайший больший** DN из доступных.

- Пример: введено DN 60, доступны [50, 65, 80] → берётся 65
- Если DN больше максимального — возвращается ошибка "DN не найден"

## 10. ПРОЦЕСС ПОЛУЧЕНИЯ ЦЕНЫ ПОЛЬЗОВАТЕЛЕМ

1. Пользователь нажимает кнопку "Показать цену" на карточке прибора
2. Открывается модальное окно с полями: телефон, email, галочка согласия
3. На email отправляется 6-значный код верификации
4. Пользователь вводит код
5. После подтверждения кода выполняется расчёт цены на основе параметров поиска (DN, давление, точность)
6. Результат отображается в модальном окне
7. Автоматически отправляется письмо на sktb_razvitie1@turbo-don.ru с данными о запросе

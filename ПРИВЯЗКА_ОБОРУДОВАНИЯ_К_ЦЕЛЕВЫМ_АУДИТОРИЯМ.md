## 5. МЕХАНИЗМ РАБОТЫ ПРИВЯЗКИ

### 5.1. Фильтрация по ID оборудования
При выборе категории система передает параметр `allowedEquipmentIds` в API поиска, который содержит список разрешенных ID оборудования для данной категории.

**Пример:**
- Категория `extraction` → `allowedEquipmentIds: "ufg2"`
- Категория `processing` → `allowedEquipmentIds: "ufg2,tfgh1,tfgs1"`

### 5.2. Автозаполнение параметров
Каждая категория имеет предустановленные значения параметров (`defaults`), которые автоматически заполняют форму ввода параметров при выборе категории.

### 5.3. Специальные теги для 1С
Некоторые категории имеют поле `specialTag`, которое добавляется в JSON при отправке заявки в систему 1С для дополнительной маркировки типа заявки.

---

## 6. ПРИМЕЧАНИЯ

1. **Категория "Другое"**: Для категорий `other` (как для газов, так и для жидкостей) не применяется фильтрация по ID — показываются все доступные расходомеры соответствующей среды.

2. **Категория "Технические газы"**: Открывает форму заявки напрямую, минуя форму подбора параметров, так как требует индивидуального подхода.

3. **Автозаполнение**: Значения автозаполнения являются рекомендуемыми и могут быть изменены пользователем вручную.

4. **Фильтрация**: После фильтрации по ID оборудования дополнительно применяется фильтрация по техническим параметрам (давление, температура, расход, точность, диаметр, плотность).

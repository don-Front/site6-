# Инструкция по настройке отправки Email для заявок на блочно-модульные изделия

## Быстрый старт

1. Откройте файл `email_config.py`
2. Измените параметры в соответствии с вашим email провайдером
3. Установите `SEND_EMAIL = True`
4. Перезапустите приложение

---

## Подробная настройка

### Шаг 1: Выбор SMTP сервера

В зависимости от вашего email провайдера, используйте следующие настройки:

#### Gmail
```python
SMTP_SERVER = "smtp.gmail.com"
SMTP_PORT = 587
```

**Важно для Gmail:**
- Необходимо использовать **Пароль приложения** (App Password), а НЕ обычный пароль
- Как создать пароль приложения:
  1. Перейдите в настройки аккаунта Google: https://myaccount.google.com/
  2. Безопасность → Двухэтапная аутентификация (должна быть включена)
  3. Пароли приложений → Создать новый
  4. Выберите "Почта" и "Windows компьютер"
  5. Скопируйте сгенерированный пароль (16 символов)

#### Yandex
```python
SMTP_SERVER = "smtp.yandex.ru"
SMTP_PORT = 587
```

#### Mail.ru
```python
SMTP_SERVER = "smtp.mail.ru"
SMTP_PORT = 587
```

#### Корпоративный Exchange Server
```python
SMTP_SERVER = "smtp.your-company.com"
SMTP_PORT = 587  # или уточните у IT отдела
```

### Шаг 2: Укажите данные отправителя

```python
SENDER_EMAIL = "your-email@gmail.com"  # Ваш email
SENDER_PASSWORD = "abcd efgh ijkl mnop"  # Пароль приложения (для Gmail)
```

### Шаг 3: Укажите получателя

```python
RECEIVER_EMAIL = "sales@your-company.ru"  # Email для получения заявок
```

Опционально можно добавить дополнительных получателей в копию:

```python
CC_EMAILS = ["manager@your-company.ru", "director@your-company.ru"]
```

### Шаг 4: Активируйте отправку

```python
SEND_EMAIL = True
```

### Шаг 5: Перезапустите приложение

После изменения настроек перезапустите сервер:

```powershell
# Остановите текущий процесс (Ctrl+C)
# Запустите заново:
cd app
python -m uvicorn main:app --reload --host 192.168.2.127 --port 8000
```

## Проверка работы

1. Откройте главную страницу приложения
2. Нажмите на карточку "Блочно-модульные изделия"
3. Заполните форму и прикрепите тестовый файл
4. Нажмите "Отправить заявку"
5. Проверьте:
   - В логах сервера должно быть сообщение: `✅ Email успешно отправлен`
   - На указанный email должно прийти письмо с заявкой

## Устранение проблем

### "Ошибка аутентификации" или "Username and Password not accepted"

**Для Gmail:**
- Убедитесь, что используете пароль приложения, а не обычный пароль
- Проверьте, что двухэтапная аутентификация включена

**Для других провайдеров:**
- Проверьте правильность email и пароля
- Возможно требуется разрешить доступ для "менее безопасных приложений" в настройках email

### "Connection refused" или "Timeout"

- Проверьте правильность SMTP сервера и порта
- Убедитесь, что брандмауэр не блокирует исходящие соединения на порт 587
- Попробуйте использовать порт 465 с SSL:
  ```python
  SMTP_PORT = 465
  # И измените в коде: server = smtplib.SMTP_SSL(SMTP_SERVER, SMTP_PORT)
  ```

### Email не приходит, но ошибок нет

- Проверьте папку "Спам" в почтовом ящике
- Убедитесь, что RECEIVER_EMAIL указан правильно
- Проверьте логи сервера на наличие предупреждений

### "Email не отправлен: Конфигурация email не настроена"

- Убедитесь, что файл `email_config.py` находится в папке `app/`
- Проверьте, что установлен `SEND_EMAIL = True`

## Дополнительная информация

### Где хранятся заявки?

Все заявки сохраняются локально в папке:
```
app/received_data/modular_requests/
```

Структура:
- `YYYYMMDD_HHMMSS_filename.ext` - загруженный файл ТЗ
- `YYYYMMDD_HHMMSS_data.json` - данные заявки в формате JSON

Даже если email не отправился, данные сохраняются локально!

### Безопасность

**Важно:**
- Никогда не публикуйте файл `email_config.py` с реальными паролями в публичных репозиториях
- Используйте пароли приложений вместо основных паролей
- Рассмотрите использование переменных окружения для хранения чувствительных данных

### Пример использования переменных окружения (опционально)

Вместо хардкода паролей в `email_config.py`, можно использовать:

```python
import os

SENDER_EMAIL = os.getenv("EMAIL_SENDER", "your-email@gmail.com")
SENDER_PASSWORD = os.getenv("EMAIL_PASSWORD", "")
RECEIVER_EMAIL = os.getenv("EMAIL_RECEIVER", "recipient@example.com")
```

И установить переменные окружения в Windows:
```powershell
$env:EMAIL_SENDER = "your-email@gmail.com"
$env:EMAIL_PASSWORD = "your-app-password"
$env:EMAIL_RECEIVER = "recipient@example.com"
```
